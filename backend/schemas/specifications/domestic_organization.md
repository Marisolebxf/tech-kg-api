# 国内机构数据表规范

本规范依据第三方最新《要素库设计方案》生成，覆盖全部 16 张表和 201 个字段。

## 建模约定

1. 表名、字段名、字段长度和字段顺序保持第三方规范。
2. “是否有空值”为“否”的字段使用 `NOT NULL`。
3. “是否索引”为“是”的字段建立普通索引。
4. 原规范没有“是否主键”列，因此当前不擅自建立主键或唯一约束。
5. 字段描述明确写有“唯一标识”的字段属于候选主键，待业务确认后升级约束。
6. 原规范没有提供数据样例的字段暂时留空，后续按统一示范数据规则补充。

## 类型映射

| 第三方类型 | MySQL 类型 |
|---|---|
| 字符型 | `VARCHAR(n)` |
| 文本型 | `TEXT` |
| 数字型 `20` | `BIGINT` |
| 数字型 `20,2` | `DECIMAL(20,2)` |
| 日期型 | 业务日期使用 `DATE`；`created_time`、`updated_time` 使用 `DATETIME` |
